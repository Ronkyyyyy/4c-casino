# 🎰 四字熟語カジノ (Yojijukugo Casino)

**「画数密度」を見極めろ。ハイリスク・ハイリターンな漢字ベッティングゲーム。**

四字熟語を構成する漢字の「画数」を予想し、チップを賭けるWebブラウザゲームです。
単に画数が多い字を当てるだけではありません。**「読み仮名が長いほどコストがかかる」**という独自のルールにより、低コストで高配当を狙う戦略眼と、バッティング（重複）を回避する運が試されます。

## 🎮 ゲーム概要

プレイヤーは所持金（500チップ）を元手に、全10ラウンドを戦い抜きます。
読み仮名（ひらがなカード）を選択して漢字を作り、その画数がオッズ（倍率）となります。

### 特徴
- **画数密度 (Stroke Density)**: 「読みが短い（1〜2文字）」かつ「画数が多い」漢字が最強の配当を生みます。
- **バッティング没収**: 同じ画数の漢字を選んでしまった場合、配当は**ゼロ**。他のプレイヤー（架空）と被らない「孤独な数字」を探す心理戦です。
- **カジノ演出**: バッティング確定時にリールが赤く染まる「絶望のタメ」や、ドラムロール演出で臨場感を煽ります。

## 📜 ルール

### 1. BETフェーズ
カードが裏向きの状態で配られます。この時点では熟語はわかりません。
自信に合わせて **BET額（単価）** を決定します（10 / 50 / 100 / 500）。

### 2. ACTIONフェーズ
カードが表になり、10秒のカウントダウンが始まります。
ひらがなの並びを見て、**「1つの漢字になる範囲」**を選択します。

> **💸 コストの支払い**
> 選択した **カード枚数 × BET額** が、即座に所持金から引かれます。
> * 例：BET 100で「いっしょう（5枚）」を選択 → **500チップ支払い**
> * 例：BET 100で「けん（2枚）」を選択 → **200チップ支払い**
>
> ※読みが長い漢字は、当たっても利益が出にくい（コスパが悪い）設計です。

### 3. JUDGEMENTフェーズ
結果判定が行われます。

* **WIN（的中）**:
    **「BET額 × 漢字の画数」** のチップが払い戻されます。
* **INVALID（区切りミス）**:
    払い戻しなし（コスト丸損）。
* **BAT OUT（バッティング）**:
    選んだ漢字の画数が、その熟語内の他の漢字と重複していた場合、**払い戻しなし（没収）**。
    リールが赤く染まり、チップは闇に消えます。

### 👑 ボーナスルール
その熟語の中で、**「重複していない中で最大の画数」**を持つ漢字を当てた場合、**JACKPOTボーナス（配当1.2倍）** が適用されます。
※本来の最大画数がバッティングしていた場合、次点の画数がボーナス対象に繰り上がります。

### 💀 ペナルティ
制限時間内に選択しなかった場合（TIME OVER）、**「その熟語の全文字数 × BET額」** が強制徴収されます。
（最も高い買い物をさせられた上で商品なし、という重罰です）

## 📝 問題データの管理

`script.js` 内の `PROBLEM_POOL` 配列で管理します。
Googleスプレッドシート等でデータを管理し、以下のJSON形式に変換して追記してください。

```javascript
{
    text: "満身創痍",
    hiragana: ["ま", "ん", "し", "ん", "そ", "う", "い"],
    solutions: [
        { range: [0, 1], kanji: "満", strokes: 12 }, // Batting!
        { range: [2, 3], kanji: "身", strokes: 7 },
        { range: [4, 5], kanji: "創", strokes: 12 }, // Batting!
        { range: [6], kanji: "痍", strokes: 11 }     // Jackpot Candidate
    ]
},

```

## ⚠️ 注意事項

* **画数判定**: 一般的な常用漢字の画数を基準としています。
* **バッティング判定**: 違う漢字であっても「画数」が同じであればバッティング（没収）とみなされます。
